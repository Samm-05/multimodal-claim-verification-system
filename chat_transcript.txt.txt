PHASE 1: REQUIREMENT ANALYSIS
============================================================

Objective:
Build an automated system capable of verifying damage claims using:

1. Images
2. User Conversations
3. User History
4. Evidence Requirements

Supported Objects:
- Car
- Laptop
- Package

Required Outputs:
- supported
- contradicted
- not_enough_information

Key Insight:

Images are the primary source of truth.

User history may contribute risk context but must never override clear visual evidence.

Decision:
Adopt a modular multi-agent architecture to separate responsibilities and improve explainability.

============================================================
PHASE 2: SYSTEM DESIGN
============================================================

Selected Architecture:
Multi-Agent Pipeline

Agent Design:

1. Claim Extraction Agent
2. Vision Analysis Agent
3. Image Quality Agent
4. Evidence Validation Agent
5. Risk Assessment Agent
6. Decision Agent

Reasoning:

A multi-agent design allows each stage of claim verification to be independently evaluated, tested, and improved.

Benefits:

- Modular
- Scalable
- Explainable
- Maintainable
- Production-Oriented

============================================================

Goal:

Extract structured claim information from customer conversations.

- Identify object part
- Normalize claim information

============================================================
PHASE 5: IMAGE QUALITY AGENT
============================================================

Goal:

Determine whether submitted images are suitable for automated review.

Checks:

- Blur detection
- Low light
- Cropping
- Obstruction
- Wrong angle

Generated Risk Flags:

- blurry_image
- cropped_or_obstructed
- low_light_or_glare
- wrong_angle

Output:

valid_image
risk_flags

============================================================
PHASE 6: EVIDENCE VALIDATION AGENT
============================================================

Goal:

Verify that submitted images satisfy evidence requirements.

Input:

Responsibilities:

- Check visual sufficiency
- Verify required object visibility
- Validate claim evidence

Output:

evidence_standard_met_reason

============================================================
PHASE 7: RISK ASSESSMENT AGENT
============================================================

Goal:

Analyze historical claim behavior.

Input:

user_history.csv

Factors:

- Past claim count
- Rejected claims
- Manual review history
- Claim frequency

Generated Flags:

- user_history_risk
- manual_review_required

Important Design Decision:

Risk signals never override visual evidence.
